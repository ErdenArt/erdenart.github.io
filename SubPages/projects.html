Chess Movement Algorithm

The chess movement algorithm is designed to determine the valid moves for each piece on the chessboard based on the rules of chess (or even your own). Each piece has its own unique movement pattern, and the algorithm takes into account the current position of the pieces, the state of the board, and any special rules such as castling, en passant, and pawn promotion.

To implement this algorithm, we first need to define a base class for chess pieces and then create subclasses for each specific piece type (Pawn, Rook, Knight, Bishop, Queen, King). For example in my game "Sword of magic", that I am developing in GameMaker, I have created a struct that every piece inherits from:

  1. "Player":

  4.   sprite: spr_player,
  5.   health: 20,
  6.   movement: [0,0,1,0,0
  7.              0,1,1,1,0,
  8.              1,1,0,1,1,
  9.              0,1,1,1,0,
  10.              0,0,1,0,0]
  11.   ...
  12. },

In this example, the "movement" array defines the movement pattern for the piece. The values in the array represent the possible moves relative to the piece's current position. A value of 1 indicates a valid move, while a value of 0 indicates an invalid move. The center of the array (the position of the piece itself) is represented by the middle element.

When calculating valid moves, the algorithm iterates through the "movement" array, checking each position relative to the piece's current location on the board.
